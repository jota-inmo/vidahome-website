## 🔄 Cambios Implementados

### 1. **Formulario de Vender - Mejoras UX** (commit: `380f45f`, `36e1dd9`)

#### a) Scroll automático entre pasos
- **Archivo**: `src/app/[locale]/vender/page.tsx`
- **Cambio**: Se añadió función `scrollToTop()` que hace scroll suave al cambiar de paso
- **Comportamiento**:
  - Al avanzar con "Siguiente" → scroll arriba
  - Al retroceder con "Volver" → scroll arriba
  - Al saltar automáticamente al paso de revisión → scroll arriba
- **Motivo**: Mejorar la intuitividad al navegar el formulario multipaso

#### b) Reorganización de tipos de propiedad en 4 categorías
- **Archivos**: 
  - `src/types/sell-form.ts`
  - `src/app/[locale]/vender/components/PropertyTypeStep.tsx`
  
- **Cambio**: Implementación de interfaz con 4 tarjetas principales:
  1. **Residencial** - Piso, Apartamento, Casa/Chalet rústico, Casa aislada, Adosado, Pareado, Adosado esquinero
  2. **Comercial** - Local comercial, Oficina, Almacén
  3. **Terreno** - Terreno urbano, Terreno urbanizable, Terreno no urbanizable, Terreno rústico
  4. **Otros** - Garaje, Aparcamiento, Trastero, Otro tipo

- **UX**: Al seleccionar una categoría se despliegan sus tipos específicos debajo
- **Etiquetas actualizadas**:
  - "Apartamento de lujo" → "Apartamento"
  - "Chalet/Casa aislada" → "Casa/Chalet rústico"
  - Añadido: "Terreno no urbanizable" como 4º tipo de terreno

#### c) Ajuste del flujo según tipo de propiedad
- **Archivo**: `src/app/[locale]/vender/page.tsx`
- **Cambio**: Ahora cuando se encuentra un inmueble desde Catastro:
  - Si es **piso o apartamento** → pasa por el paso 4 (Detalles) antes de Revisión
  - Para **otros tipos** → salta directo a Revisión (paso 5)
- **Motivo**: Recoger planta y puerta solo cuando sea necesario

### 3. **Sistema de Releases Semver** (commit: `d8a6989`)

#### Automatización de versionado Git
- **Archivos**: `package.json`, `README.md`
- **Nuevos comandos**:
  ```bash
  npm run release:patch  # 0.1.0 -> 0.1.1 (fixes)
  npm run release:minor  # 0.1.0 -> 0.2.0 (features)
  npm run release:major  # 0.1.0 -> 1.0.0 (breaking changes)
  ```
- **Funcionamiento**: Cada comando:
  1. Actualiza la versión en `package.json`
  2. Crea un commit automático
  3. Crea un tag Git anotado
  4. Hace push del commit y el tag a `origin/main`
